Coverter

Converts a State-transitions Table for the Turing Machine emulator into a series of byte codes to load into the EEPROM memory of the emulator.

Expand the instructions below when using it for the first time.

Step by step instructions

Manually enter or simply paste a State Table into the box below.

The State Table might look like the following

##State  Symbol   Print Operation Next State
   s1      0        -       -            Halt
   s1      1        0       R            s2
   s2      0        0       R            s3
   s2      1        1       R            s2
   s3      0        1       L            s4
   s3      1        1       R            s3
   s4      0        0       L            s5
   s4      1        1       L            s4
   s5      0        1       R            s1
   s5      1        1       L            s5
  • Everything after the first # in a line is ignored, like the headings above.
  • There should be an entry per line, where the various parts are separated by one or more blank spaces or tabs. There is no particular formatting required, the following table, more compact than the example above, is just as good. 
    # Overflow left
A 0 1 L B
A 1 1 L B
B 0 - L B
B 1 0 L B
  • All entries are case-insensitive, an l and an L are just the same.
  • The columns are:
    1. State: A label, any characters except # and spaces are valid. There should be two rows with the same label, one for each State. Labels are also case-insensitive so a label like A is the same as a. A label cannot spell the keyword halt as that is reserved.
    2. Symbol: Either a 0 or a 1 for the Symbol read from the tape.
    3. Print, what and whether to print on the tape, it can be a 0, a 1 or - which means don't print anything.
    4. Move, whether to move the tape and in which direction. Can be L for left, R for right and - for don't move.
    5. Next State: Which state to go next. It should match one of the labels on the first column, or can be the word Halt to stop the execution
  • When the table is keyed in or pasted from elsewhere (the most common case), press the Convert button below the table to process.
  • The results will appear below, along with instructions.
  • Either of the sample tables above can be copied and then pasted, just to try it out.

The box below contains a sequence of numbers, the byte codes, that can be pasted into the EEPROM in the emulator. They have already been copied into the clipboard.
